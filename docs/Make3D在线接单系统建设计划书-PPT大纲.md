# Make3D 在线接单系统建设计划书 - PPT 大纲版

## 1. 项目目标
- 建立在线 3D 打印接单、自动报价辅助、后台订单管理的一体化系统
- 降低微信沟通成本，沉淀客户、订单、文件和切片报价数据
- V1/V2 阶段坚持安全边界：保存文件、自动报价辅助、人工最终确认

## 2. 目标客户
- 学生与毕业设计用户
- 创客与个人开发者
- 小微企业与工作室
- 西安本地客户

## 3. 整体业务流程
- 访问首页 -> 登录/注册 -> 在线报价 -> 上传模型 -> 自动报价/人工确认 -> 提交订单 -> 管理员复核 -> 状态流转 -> 交付完成

## 4. 前台页面结构
- 首页
- 在线报价 /quote
- 注册、登录、找回密码
- 用户中心 /account
- 我的订单与订单确认页

## 5. 会员系统设计
- 手机号唯一账号
- 密码 hash 保存
- customer_session 与管理员 session 分离
- 未登录可浏览报价布局，但上传、报价、提交全部禁用

## 6. 自动报价系统设计
- STL 调用 PrusaSlicer CLI
- 解析 G-code 中耗材重量和打印时间
- 计算材料费、工时费、包装费、配送费和预计交货期
- STEP/STP/3MF 暂时人工确认

## 7. 后台订单管理设计
- 管理员登录、订单列表、订单详情、文件下载、状态修改、物流与备注、手动测试切片报价

## 8. 订单状态流转
- 待确认 -> 待付款 -> 已付款 -> 排产中 -> 打印中 -> 后处理 -> 已发货 -> 已完成
- 任意阶段可取消，状态变更记录日志

## 9. 微信公众号通知规划
- 后续绑定 OpenID
- 待付款、打印中、已发货、已完成等关键节点推送
- 与邮件通知形成互补

## 10. 文件上传与切片报价流程
- 登录校验 -> 文件校验 -> 保存 uploads -> STL 自动切片 -> 保存 slice_jobs -> 失败转人工确认

## 11. 权限与防滥用设计
- 未登录 API 返回 401
- 客户登录失败分级封禁
- 上传限流
- 切片超时和并发限制
- 不执行用户上传模型文件

## 12. 数据表/核心结构
- customers
- orders
- files
- slice_jobs
- order_status_logs
- auth_blocks

## 13. 分阶段开发计划
- V1.0 基础接单
- V1.1 多文件与报价增强
- V1.2 会员与订单中心
- V2 自动切片报价
- V2.x 通知、支付、排产和运维增强

## 14. 当前已完成内容
- 基础框架、上传、后台、会员、订单中心、自动报价、Docker 部署、PrusaSlicer 容器支持
