## Diagnosis

Current proxy mode:

```text
Windows system/user proxy is enabled through verge-mihomo on 127.0.0.1:7897.
WSL has no proxy environment variables and no apt proxy config.
WSL can reach the Windows host proxy service through 172.30.64.1.
```

Current WSL mode:

```text
Default WSL2 NAT-style network.
No explicit mirrored networking config found.
```

Current DNS state:

```text
WSL generated resolv.conf points to 10.255.255.254.
Ubuntu and related package domains resolve to 198.18.0.x fake-ip addresses.
Direct WSL access to those addresses times out.
```

Root cause hypothesis:

```text
WSL is receiving fake-ip DNS answers but does not have a complete working proxy path for those fake-ip destinations.
Explicit proxy access to the Windows proxy is reachable, but current proxy routing for archive.ubuntu.com/security.ubuntu.com does not complete the TLS connection.
```
