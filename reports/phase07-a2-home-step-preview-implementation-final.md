## Root Cause And Preview Repair

The quote preview mixed two independent artifact roles. A browser `File` preview could later be replaced by a draft/download URL while React effect cleanup disposed the reusable WebGL canvas with `forceContextLoss`. Slicing completion also exposed G-code state near the model state, making it possible for an unsuitable artifact path to become a preview candidate.

The repair:

- Added a stable preview-source selector where a current browser `File` remains authoritative.
- Explicitly rejects `.gcode` URLs as model preview sources.
- Keeps model preview URL and generated G-code path as separate fields and flows.
- Removed forced WebGL context loss from thumbnail cleanup when the canvas is reused.
- Uses a validated derived STL URL for STEP/STP preview after conversion.
- Preserves source filename and draft-file identity across slicing and refresh.

## STEP And Geometry Implementation

- Added case-insensitive `.stl`, `.step`, and `.stp` validation.
- Added STEP Part 21 checks for `ISO-10303-21`, `HEADER`, `DATA`, recognizable entities, `ENDSEC`, and `END-ISO-10303-21`.
- Added binary/ascii STL structural validation and source SHA-256 calculation.
- Added a confined STEP-to-STL derived-artifact pipeline using an argument-array process invocation, bounded timeout/output, `.part.stl`, atomic publication, and size/SHA verification.
- Added shared inclusive auto-quote geometry rules: every model dimension must be between 10 mm and 300 mm.
- Server-side STL/derived-STL dimensions are authoritative; temporary STEP estimates are not presented as verified dimensions.
- Added nullable source/conversion provenance columns to the application schema code for `quote_draft_files` and `files`. No production schema migration was run.
