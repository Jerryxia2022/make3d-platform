# Phase05-H-A Lease Contract Correction Final

Date: 2026-07-15

## Scope

This phase corrected only the Worker Slicing lock/lease contract and revalidated the local closed loop.

No production deployment, SSH production access, production database access, production token change, real customer file, real customer slicing task, order change, quote change, payment change, WeChat Pay change, upload-limit change, or slicing systemd service creation was performed.

## 1. Root Cause

The Phase05-H-A local closed-loop run succeeded, but its initial Worker Slicing lease did not match the frozen contract.

Observed mismatch:

```text
lease_expires_at_ms: 1784104861141
lease_renewed_at_ms: 1784104561141
delta: 300000 ms
frozen contract: 120000 ms
```

Cause:

```text
src/backend/workerSlicingJobs.ts used DEFAULT_LEASE_DURATION_MS = 5 * 60 * 1000
src/backend/workerSlicingJobs.ts used DEFAULT_LOCK_DURATION_MS = 5 * 60 * 1000
src/app/api/worker/slicing/jobs/[id]/lease/route.ts separately hardcoded 120000
```

The initial lock path therefore used 300000 ms, while the lease route used 120000 ms.

## 2. New Lease Contract

Single source:

```text
WORKER_SLICING_LEASE_DURATION_MS = 120000
```

Applied behavior:

```text
initial lease_expires_at_ms = now_ms + 120000
initial lock_expires_at_ms = now_ms + 120000
renewal new_expiry = MAX(current_expiry, now_ms + 120000)
renewal updates lease_renewed_at_ms
expired renewal returns 409 LEASE_EXPIRED
```

Phase03 local file sync Worker lock recovery remains unchanged at its existing 15 minute timeout.
